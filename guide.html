從本地開發到雲端發佈的完整實作流程紀錄

📦 階段 0：專案目錄規範

為了讓網址專業且檔案整齊，我們採用業界常用的 docs 目錄法：

  • /docs/：存放正式上線的網頁程式（index, css, js 等）。
  • /第一版試做/：存放開發初期的備份。
  • 根目錄：存放 Git 設定檔與說明文件。
💡 優點： 這種佈局能讓 GitHub 網址最乾淨，且檔案管理層次分明。

🌐 階段 1：GitHub 網站託管 (Hosting)

Step 1. 建立倉庫與存檔

  1. 打開 GitHub Desktop，選擇 File > Add Local Repository。
  2. 指向你的專案資料夾。若提示非倉庫，點擊連結建立專案。
  3. 在左下角 Summary 填入 初始化版本，點擊 Commit to main。
  4. 點擊上方 Publish Repository。
  5. 重要： 務必取消勾選 Keep this code private。

Step 2. 開啟 GitHub Pages 免費網址

  1. 到 GitHub 網頁端的專案頁面，點選 Settings。
  2. 點擊左側 Pages。
  3. 在 Branch 區塊選擇 main，右邊的資料夾路徑選成 /docs 魔法設定。
  4. 點擊 Save。大約 1 分鐘後你的網址就生效了！
您的網址：https://[Username].github.io/[Project-Name]/

🔥 階段 2：Firebase 雲端資料庫

Step 1. 建立資料中心

  1. 前往 Firebase Console 建立新專案。
  2. 在 Firestore Database 點擊建立，選擇 測試模式 (Test mode) 啟動。
  3. 位置選取 asia-east1 (台北)。

Step 2. 獲取 API 金鑰

  1. 進入 專案設定，在 Web 應用程式 () 註冊。
  2. 複製 firebaseConfig 內的物件內容。
  3. 打開 docs/firebase-config.js，貼入剛剛複製的金鑰資訊。
